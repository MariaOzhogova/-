CSRF PoC — Тестирование защиты

Эта страница раздаётся с localhost:8080, поэтому fetch-запросы работают корректно.

Сначала залогинься на главной странице, потом запускай тесты.

Результаты появятся здесь...

Тест 1: POST без CSRF-токена (атака)

Отправляем POST на /api/save-lab без токена. Ожидание: 403 Forbidden

Тест 2: POST с поддельным CSRF-токеном (атака)

Отправляем POST с фейковым токеном. Ожидание: 403 Forbidden

Тест 3: POST с пустым CSRF-токеном (атака)

Отправляем POST с пустой строкой вместо токена. Ожидание: 403 Forbidden

Тест 4: GET вместо POST

Пробуем GET на POST-эндпоинт. Ожидание: 405 Method Not Allowed

Тест 5: Доступ к защищённой странице без cookies

Проверяем, перенаправляет ли / на логин. Ожидание: 302 Redirect

Тест 6: POST с валидным CSRF-токеном (легитимный запрос)

Берём настоящий токен со страницы / и отправляем POST. Ожидание: 200 OK
